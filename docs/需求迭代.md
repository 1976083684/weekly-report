### 修复Bug

| 文件 | 问题描述 |
|------|----------|
| `src/middleware.ts` | **Edge Runtime 500错误** — 中间件导入 `@/lib/auth` 导致 PrismaAdapter + @prisma/client 在 Edge 环境无法加载，所有请求返回500。修复：拆分为 `auth.config.ts`（Edge安全）+ `auth.ts`（Node.js），中间件使用 NextAuth(authConfig) |
| `src/app/api/ai/optimize/route.ts` | **AI优化按钮报500** — Prisma Client未重新生成导致 `aIModel` 属性 undefined。清除 node_modules/.prisma/client 后重新生成 |
| `src/components/ai/AiOptimizeButton.tsx` | **AI优化按钮报"网络请求失败"** — 旧 dev server 进程未包含 `/api/ai/optimize` 路由，fetch 返回 HTML 导致 JSON 解析失败 |
| `src/app/(dashboard)/dashboard/page.tsx` | **活跃度日期偏差** — `toISOString().split("T")[0]` 在 UTC+8 时区可能差一天（如周一显示为周日日期）。使用 getFullYear/getMonth/getDate 的本地日期替代 |
| `src/app/(dashboard)/dashboard/page.tsx` | **isFuture 判断不稳定** — Date 对象比较 `day.date > new Date()` 含时间分量。改为字符串日期比较 `dateStr > toLocalDateStr(new Date())` |
